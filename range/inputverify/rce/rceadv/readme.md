# 命令执行实战 靶场

## 靶场简介

天积网络诊断中心提供在线Ping网络连通性检测功能。该系统在处理用户输入时存在命令注入漏洞，可通过该漏洞完成反弹shell、创建用户、开放端口等实战操作。

## 学习目标

理解命令执行漏洞在真实渗透测试中的三大核心利用方向：

- **权限获取**：通过反弹shell获取目标系统交互式控制权
- **权限维持**：通过创建后门用户实现持久化访问
- **持久化/横向移动**：通过开放高危端口服务实现持续控制

## 任务要求

利用命令注入漏洞完成以下三个实战挑战。每完成一个操作后在"成就验证"卡片中点击验证按钮确认

## 操作提示

- 观察Ping网络诊断工具的输入框，思考用户输入是如何被处理的
- 你需要自己在本地搭建监听服务器来接收反弹shell
- 注入的反弹shell命令**必须在后台运行**，否则会导致诊断工具超时无响应
  - Windows: 使用 `start /B` 前缀
  - Linux: 命令末尾加 `&`
- 当前系统环境信息已在任务提示中显示，注意选择适合的命令注入方式
- 某些操作可能需要多条命令分步执行
- 完成操作后记得在"成就验证"中点击验证

## 运行环境要求

Web服务器需以管理员/root权限运行：

- Windows: phpstudy默认以管理员权限运行
- Linux: 需确保Web进程有创建用户和设置计划任务的权限

## ⚠️ Windows风险提示

成就三要求使用计划任务开启RDP端口，请在实验完成后删除计划任务，避免端口真的被打开。
